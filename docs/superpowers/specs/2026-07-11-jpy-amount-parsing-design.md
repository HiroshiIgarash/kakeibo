# メール金額パーサー JPY 表記対応 設計書

日付: 2026-07-11
ステータス: 承認待ち

## 背景・目的

三井住友カード（Vpass）の利用通知メールのうち、海外加盟店経由の円建て決済
（実例: GOOGLE*YOUTUBE MEMBER の YouTube メンバーシップ）は金額が
`◇利用金額：990.00 JPY` のように **数値 + JPY** で記載される。

現行の `AMOUNT_RE` は `…円` 表記のみ対応のため、これらは毎月 parse_error →
失敗メールとなり手入力が必要。JPY は円そのものなので自動取り込みに変更する。

USD 等の他通貨は換算が必要なため、従来どおり失敗メール（`amountRaw` ヒント付き）
＋手入力フローを維持する。

## 変更対象

`apps/web/src/lib/email-parser.ts` の `parseSmbcEmail` のみ。
`extractSmbcFields`（失敗メールのプリフィル用部分抽出）は変更しない。

## 仕様

### 金額抽出の優先順

1. 既存 `AMOUNT_RE`（`利用金額：990円` 形式）— 変更なし
2. 新規 `AMOUNT_JPY_RE`（円表記でマッチしなかった場合のみ試行）:

```
/(?:ご)?利用金額[：:]\s*([\d,]+)(?:\.(\d+))?\s*JPY/
```

### JPY 表記の解釈

- 整数部: カンマ除去して整数化（`1,990.00 JPY` → 1990）
- スペース有無は不問（`990.00 JPY` / `990JPY` とも可）
- 小数部が **すべてゼロ**（`.00`）または無し → その整数額の円として成功
- 小数部が **非ゼロ**（例 `990.50 JPY`）→ parse_error。
  金額を勝手に丸めない。従来どおり失敗メール＋手入力にフォールバック
- `JPY` 以外の通貨コード（USD 等）→ 従来どおり parse_error

### エラーメッセージ

既存の `抽出失敗: 利用金額` を維持（欠落フィールド列挙の仕組みは変更しない）。
小数部非ゼロの JPY も「利用金額の抽出失敗」として扱う。

## テスト（`apps/web/src/lib/email-parser.test.ts`）

- `◇利用金額：990.00 JPY` → ok, amount=990（既存の「失敗する」期待テストを反転）
- `◇利用金額：990JPY` → ok, amount=990
- `◇利用金額：1,990.00 JPY` → ok, amount=1990
- `◇利用金額：990.50 JPY` → parse_error
- `◇利用金額：9.99 USD` → parse_error（既存挙動の回帰確認）
- 円表記（`990円` / `1,990 円`）の既存テストが全て green のまま

`queries.test.ts` 等の「失敗メール一覧に JPY の amountRaw ヒントが出る」テストは、
DB に failed 状態で保存済みのレコードを扱うものなので変更不要。

## スコープ外

- 既存の失敗メールの自動再パース（溜まっている分は既存の解決 UI で手入力。
  本変更のデプロイ以降に届くメールから自動取り込みされる）
- USD 等の外貨換算
